# Entity Lifecycle Events

A Spring Boot mini-project demonstrating how to hook into **JPA entity lifecycle events** (`@PrePersist`, `@PostLoad`, `@PreUpdate`, `@PostPersist`, `@PostRemove`) to run business logic automatically at specific points in an entity's persistence lifecycle — without any manual calls in the service layer.

---

## What This Project Demonstrates

JPA defines five lifecycle callback annotations that fire at well-defined moments during an entity's life. This project uses all five on a `Product` entity:

| Annotation | When it fires | Used here for |
|---|---|---|
| `@PrePersist` | Just **before** INSERT | Auto-generate a URL slug from the product name; stamp `createdAt` / `updatedAt` timestamps |
| `@PostPersist` | Just **after** INSERT commits | Audit log entry: "product created" |
| `@PreUpdate` | Just **before** UPDATE | Audit log entry: "product updating" |
| `@PostLoad` | **After** every SELECT load | Compute the `discountedPrice` transient field from `price × (1 − discount%)` |
| `@PostRemove` | Just **after** DELETE commits | Audit log entry: "product deleted" |

Two implementation styles are demonstrated side by side:

- **Inline callbacks** — methods annotated directly on the entity class (`Product.onPrePersist()`, `Product.onPostLoad()`). Best for logic tightly coupled to the entity's own data.
- **External listener** — a separate `ProductAuditListener` class registered via `@EntityListeners`. Best for cross-cutting concerns like audit logging that could apply to multiple entity types.

### Key Behaviours Visible via the REST API

1. **POST /api/products** — the response contains a `slug` field (e.g. `"wireless-headphones"`) that was never supplied in the request body. It was generated by `@PrePersist`. The `createdAt`, `updatedAt`, and `discountedPrice` fields are also auto-populated.
2. **GET /api/products/{id}** — the `discountedPrice` field is always freshly computed by `@PostLoad`. It is never stored in the database column.
3. **PUT /api/products/{id}** — the `slug` never changes even if the product name is updated, because its column is declared `updatable = false`.

## Running with Docker Compose (Recommended)

Docker Compose starts both PostgreSQL and the Spring Boot application together. This is the recommended way to run the full stack.

### 1. Build and start all services

```bash
docker compose up --build
```

The first run downloads the `postgres:16-alpine` image and builds the application image. Subsequent runs use the Docker layer cache and start much faster.

### 2. Stop and remove containers

```bash
docker compose down
```

Data in the named volume `postgres_data` is preserved. To also delete the volume (wipe all data):

```bash
docker compose down -v
```

### 3. Rebuild after code changes

```bash
docker compose up --build
```

## Running Locally (without Docker Compose)

If you only want to run the application locally (not inside Docker), start PostgreSQL separately:

```bash
docker run -d \
  --name entity-lifecycle-db \
  -e POSTGRES_DB=productsdb \
  -e POSTGRES_USER=products_user \
  -e POSTGRES_PASSWORD=products_pass \
  -p 5432:5432 \
  postgres:16-alpine
```

Then start the application:

```bash
./mvnw spring-boot:run
```

## API Usage (curl Examples)

All examples assume the application is running on `http://localhost:8080`.

### Create a product

The `slug`, `createdAt`, `updatedAt`, and `discountedPrice` are **never sent by the client** — they are all populated automatically by JPA lifecycle callbacks.

```bash
curl -X POST http://localhost:8080/api/products \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Wireless Headphones",
    "description": "Noise-cancelling over-ear headphones",
    "price": 99.99,
    "discountPercent": 20
  }'
```

Example response — notice the auto-generated `slug` (`"wireless-headphones"`) and the computed `discountedPrice` (`79.99`):

```json
{
  "id": 1,
  "name": "Wireless Headphones",
  "slug": "wireless-headphones",
  "description": "Noise-cancelling over-ear headphones",
  "price": 99.99,
  "discountPercent": 20,
  "discountedPrice": 79.99,
  "createdAt": "2024-03-01T10:00:00Z",
  "updatedAt": "2024-03-01T10:00:00Z"
}
```

### List all products

```bash
curl http://localhost:8080/api/products
```

### List discounted products only

```bash
curl http://localhost:8080/api/products/discounted
```

### Get a product by ID

```bash
curl http://localhost:8080/api/products/1
```

### Get a product by slug

```bash
curl http://localhost:8080/api/products/slug/wireless-headphones
```

### Update a product

The `slug` remains unchanged even if the name changes (stable URLs). The `updatedAt` is refreshed. The `discountedPrice` is recomputed from the new values by `@PostLoad`.

```bash
curl -X PUT http://localhost:8080/api/products/1 \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Wireless Headphones Pro",
    "description": "Premium noise-cancelling headphones",
    "price": 129.99,
    "discountPercent": 15
  }'
```

### Delete a product

```bash
curl -X DELETE http://localhost:8080/api/products/1
```

## How the Lifecycle Callbacks Work

### `@PrePersist` — slug generation and timestamps

```java
@PrePersist
void onPrePersist() {
    // "Wireless Headphones (v2)" → "wireless-headphones-v2"
    this.slug = name.toLowerCase()
            .replaceAll("[^a-z0-9]+", "-")
            .replaceAll("-{2,}", "-")
            .replaceAll("^-|-$", "");

    Instant now = Instant.now();
    this.createdAt = now;
    this.updatedAt = now;

    computeDiscountedPrice();
}
```

Hibernate calls this automatically before every INSERT. No service or controller code touches these fields.

### `@PostLoad` — computed transient field

```java
@PostLoad
void onPostLoad() {
    computeDiscountedPrice();
}

private void computeDiscountedPrice() {
    BigDecimal factor = BigDecimal.ONE.subtract(
            BigDecimal.valueOf(discountPercent).divide(BigDecimal.valueOf(100)));
    this.discountedPrice = price.multiply(factor).setScale(2, RoundingMode.HALF_UP);
}
```

`discountedPrice` is annotated with `@Transient` — it is never stored in the database. It is recalculated fresh from the persisted `price` and `discountPercent` values every time the entity is loaded.

### External listener — `ProductAuditListener`

```java
@EntityListeners(ProductAuditListener.class) // on the entity
public class Product { ... }

public class ProductAuditListener {
    @PostPersist  public void onPostPersist(Product p) { log.info("[AUDIT] Created: {}", p.getId()); }
    @PreUpdate    public void onPreUpdate(Product p)   { log.info("[AUDIT] Updating: {}", p.getId()); }
    @PostRemove   public void onPostRemove(Product p)  { log.info("[AUDIT] Deleted: {}", p.getId()); }
}
```

JPA instantiates the listener class automatically and invokes the appropriate callback methods alongside the entity's own inline callbacks.
